Создание ссылок: - создать файл file1 и наполнить его произвольным содержимым. cat>file1 - скопировать файл file1 в file2. cp file1.txt file2.txt - создать символическую ссылку file3 на file1. ln -s file1.txt file3 - создать жесткую ссылку file4 на file1. ln file1.txt file4 - посмотреть, какие айноды у файлов. ls -ila у символичесокй id 657613 lrwxrwxrwx у жесткой id 657609 -rw-rw-r-- - удалить file1. Что стало с остальными созданными файлами? Попробовать вывести их на экран. rm file1.txt ls -ila file4 без измений, а file3 горит красным 2. Работа со ссылками: - дать созданным файлам другие, произвольные имена. mv file2.txt super2 mv file3 super3 mv file4 super4 - создать новую символическую ссылку. ln -s super2 super21 ln -s super3 super31 ln -s super4 super41 - переместить ссылки в другую директорию. Работают ли ссылки? Попробовать вывести их на экран. mkdir newFolder mv super21 newFolder/ ничего не работает 3. Права файлов: - создать два произвольных файла. cat>file1 cat>file2 - первому файлу присвоить права на чтение, запись для владельца и группы; на чтение для всех. chmod u+rw file1 chmod g+rw file1 chmod a+r file1 - второму файлу присвоить права на чтение, запись только для владельца. Сделать это в численном и символьном виде. chmod a-rw file2 chmod u+rw file2 chmod a-rw file2 chmod 600 file2 4. Создать пользователя, обладающего возможностью выполнять действия от имени суперпользователя. sudo adduser superuser sudo usermod -aG sudo superuser su testuser sudo whoami ---root
